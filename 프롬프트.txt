code-review

현재 프로젝트의 HTML/CSS/JavaScript 코드를 리뷰해줘.

리뷰 기준:
1. 파일 구조가 적절한가?
2. 함수 이름과 역할이 명확한가?
3. 중복 코드가 있는가?
4. 게임 로직과 화면 렌더링이 과도하게 섞여 있는가?
5. 버그 가능성이 높은 부분은 어디인가?

출력 형식:
- 전체 평가
- 좋은 점
- 개선할 점
- 반드시 수정할 문제
- 선택적 개선 사항

코드는 바로 수정하지 말고 먼저 리뷰 결과만 제시해줘.

review-game-logic

테트리스 게임 로직을 집중 리뷰해줘.

집중 검토 항목:
- 보드 크기 10 x 20 유지
- 현재 블록 좌표와 matrix 처리
- 좌우/아래 이동 경계값
- 회전 후 충돌 처리
- 블록 고정 시점
- 새 블록 생성 시 게임 오버 조건
- 라인 삭제 후 보드 상태

출력 형식:
| 항목 | 현재 상태 | 위험도 | 개선 제안 |

bug-hunt

현재 테트리스 코드에서 발생 가능한 버그를 찾아줘.

특히 다음을 집중적으로 봐줘:
- 충돌 판정 오류
- 회전 시 배열 인덱스 오류
- 라인 삭제 후 보드 상태 오류
- 게임 오버 조건 누락
- setInterval 중복 실행
- 키보드 이벤트 중복 등록
- 점수 계산 오류

출력은 심각도 High / Medium / Low로 분류해줘.

refactor-safe

현재 코드를 기능 변화 없이 리팩토링해줘.

제약:
- 게임 동작은 바꾸지 말 것
- 함수 단위로 역할을 분리할 것
- 중복 코드를 줄일 것
- 변수명을 이해하기 쉽게 바꿀 것
- 리팩토링 후 변경 요약을 작성할 것

수정 전후에 어떤 동작이 유지되어야 하는지도 함께 정리해줘.
